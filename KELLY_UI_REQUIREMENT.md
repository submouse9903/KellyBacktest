## 1. 핵심 결론

**켈리비율(Kelly Ratio)은 승률·평균손익비 같은 단순 지표를 대입하는 공식으로 구하면 안 됩니다.**  
반드시 **전략의 전체 거래 수익률 분포(empirical distribution)를 직접 근거**로 삼아, 아래 기대값을 **수치적으로(numerically) 최적화**해야 합니다.

$$
f^* = \arg\max_f \; E[\log(1 + fX)]
$$

여기서 $X$는 **개별 거래의 원금대비 수익률**입니다. 확률은 각 수익률의 **발생 빈도(관찰횟수 / 전체 거래횟수)**로 도출합니다.

## 2. 새롭게 추가되어야 할 UI

### 2-1. 전략 수익률 분포 플롯 (Return Distribution Plot)

**반드시 추가해야 하는 뷰입니다.**

| 항목 | 요구사항 |
|------|---------|
| **차트 종류** | 히스토그램(Histogram) + KDE(커널 밀도 추정) 오버레이 |
| **X축** | 개별 거래 원금대비 수익률 (소수로 표현, 예: -0.02, +0.03) |
| **Y축** | 빈도(Frequency) 또는 밀도(Density) |
| **상호작용** | 구간(빈)을 hover 하면 해당 구간의 빈도/확률 표시 |
| **필수 마커** | 수익률 평균(mean), 중앙값(median) 수직선 표시 |
| **추가 마커(권장)** | 켈리비율 $f^*$에 따른 최적 레버리지 지점을 별도 판넬에 표시 |

**이 플롯이 필요한 이유**
- 켈리비율은 "평균손익비와 승률이 같더라도 분포가 다륩다면 아주 많이 다르게 나옵니다"(‹켈리기준 #2›).
- 사용자가 왜 켈리비율이 그 값이 나왔는지 **분포 형태(꼬리, 스큐, 이상치)를 눈으로 확인**해야 합니다.

## 3. 백엔드 계산 로직 (필수 변경사항)

### 3-1. 데이터 준비

1. 백테스트 결과에서 **거래별 원금대비 수익률** 벡터 $R_k$를 추출합니다.
   - 예: `[-0.0006, -0.0087, ..., -0.0053, -0.0213]`
   - 퍼센트(%)가 아닌 **소수**로 변환해야 합니다.
   - 레버리지/증거금이 포함된 상태라면, **레버리지를 제거한 원금대비 수익률**로 환산해야 합니다.

2. 무위험 이자율은 당일/당기 기준으로 적용하되, 단기 트레이딩의 경우 **0으로 근사**필도 묰쇈됩니다.

### 3-2. 켈리비율 계산 — Numerical Method (권장)

**아래 가정없는 방식을 기본으로 구현하세요.**

$$
E[\log(1 + fX)] = \sum_{i=1}^{N} \frac{1}{N} \cdot \log(1 + f \cdot x_i)
$$

- $N$: 전체 거래 횟수
- $x_i$: $i$번째 거래의 수익률
- 중복된 $x_i$가 있더라도, 단순히 $1/N$ 가중치로 모두 합산하면 자동으로 경험적 확률이 적용됩니다.

**구현 예시 (Python)**

```python
import numpy as np
from scipy.optimize import minimize_scalar

def kelly_ratio(returns: np.ndarray) -> float:
    """
    returns: 거래별 원금대비 수익률 (소수), shape (N,)
    """
    returns = np.asarray(returns)
    N = len(returns)
    
    def objective(f):
        # f가 -1/최대손실을 넘지 않도록 제한 필요
        if np.any(1 + f * returns <= 0):
            return -1e10
        return -np.sum(np.log(1 + f * returns)) / N
    
    # f의 탐색 범위는 전략 특성에 따라 조정 (예: 0 ~ 10)
    result = minimize_scalar(objective, bounds=(0.0, 10.0), method='bounded')
    return result.x
```

### 3-3. 정규분포/로그노말 가정 근사식 (비권장, 참고용으로만)

분포가 매우 부드럽고 정규에 가까울 때에만 아래 식을 **참고용**으로 표시할 수 있습니다.

$$
\text{Kelly Ratio} \approx \frac{\mu - r_f}{\sigma^2}
$$

- $\mu$: 수익률 평균
- $r_f$: 무위험 이자율
- $\sigma^2$: 수익률 분산

**단, UI 상에서도 "정규분포 가정 시 근사값"이라고 명확히 라벨링해야 하며, 기본 켈리비율은 Numerical 결과를 사용해야 합니다.**

> "대부분의 경우 수익률 분포가 부드럽지 않으므로 로그노말 분포가정으로 구하면 값이 틀리게 나옵니다. 켈리비율은 굉장히 민감하므로 이런 오차가 엄청난 수익률 차이로 나타날 수 있습니다." — ‹켈리기준 #3›

## 4. 경고: 절대 하면 안 되는 방식

아래 방식은 시리즈에서 "대부분의 자금관리 프로그램이 잘못 이해하고 만들어진 것"으로 지적받는 방식입니다. **본 프로젝트에서는 사용하지 마세요.**

- 승률 $p$와 평균손익비 $b$만 넣는 단순 공식: $f = (bp - q) / b$
- "이기면 2%, 지면 1%" 같은 2~3가지 시나리오만으로 켈리비율을 계산하는 것

**이유**: 켈리의 본질은 $E[\log(1 + fX)]$의 극대화입니다. $X$가 2가지 값만 가지는 베륭누이 시행이 아니라, **실제 트레이딩에서는 연속적·불규칙한 분포**를 가지므로 전체 분포를 사용해야 정확합니다.

## 5. UI에 표시해야 할 정보 요약

켈리기준 관련 판넬/카드에는 다음 정보를 표시하세요.

1. **수익률 분포 플롯** (히스토그램 + KDE)
2. **Numerical Kelly Ratio** (기본값, 굵은 글씨)
3. **(Optional) 정규분포 근사 Kelly Ratio** — 참고용, 회색/괄호 처리
4. **통계 요약** — 평균, 중앙값, 표준편차, 왜도(Skewness), 첨도(Kurtosis)
5. **MDD 및 복리성장률 비교** — 정액베팅 수익률 대신 $E[\log(1+fX)]$ 값이 큰 전략이 더 우수함을 안내하는 문구
