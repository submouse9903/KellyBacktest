원래 주식트레이딩을 할때 저는 '챠트'만 90%이상 신뢰를 하고 트레이딩을 합니다. 가치투자니 기술투자니 뭐가 맞다 틀린다는 원론적인 토론은 뒤로 하고, machine learning을 이용하여 '애매한' 챠트를 분석할 때 필요한 변수(인자)에 대해서 요즘 고민을 하고 있습니다. 먼저 machine learning에 대하여(물론 알고 계시겠지만 ㅠㅠ) 간단히 말씀 드리면, 집합 X를 주식에 영향을 미치는 인자라고 가정하였을 때,  Training Set 1 : 볼린져 하단에 맞고 반등한 경우Training Set 2 : 볼린져 하단이 돌파하고 반등한 경우Training Set 3 : 볼린져 하단이 돌파하고 반등하지 못한 경우 이미 알고 있는 데이터들로 트레이닝을 시키고, 이 세 경우에 대하여 Support Vector Machine(Machine Learning의 한 종류)이 reasonable(구분이 확실히되는)한 plane을 제시해주는 것을 말합니다.  이렇게 될 경우, 실제 거래시 챠트가 볼린져 밴드 하단으로 접근할때 그 후에의 상황을 SVM을 이용하여 구분할수 있지 않을까 하는 생각으로, 조만간 간단한 시뮬레이션 툴을 만들어 보려고 생각합니다. 가장 고민인 부분이 집합 X에 들어가는 feature들인데, 머리속에 드는 생각으로 X = {전날 dow index 등락폭,전날의 kospi등락폭, 볼린져 상단밴드에 몇일전에 도달하였는지에 대한 일수, 볼린져 상단밴드에 몇일전에 도달하였는지에 대한 일수의 거래량/ 최근 20일 동안의 거래량, (볼린져 밴드 상단-볼린져 밴드 하단)/액면가,...} 을 생각하고 있습니다. 보통 feature는 몇 백개 몇 천개가 들어가는 경우도 있지만, 일단 작게 시작해보려고 하는데, 일반적으로 챠트의 특성을 뽑을 때 고려되는 인자들이 어떤것들이 있나요?아니면 주식에서 사용되는(제 HTS에 있는 수많은 인자들) 모든 인자를 다 넣어서 트레이닝을 해보는 방법도 있지만, 저의 삽질 시간을 줄이고자 이런 디자인 해보신 분들에게 질문 드려봅니다. 투표는 표시되지 않습니다.
