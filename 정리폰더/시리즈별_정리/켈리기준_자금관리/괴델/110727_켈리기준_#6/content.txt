선물시스템트레이딩 전략은 포트폴리오로 운영하는 것이 중요합니다.. 분산투자효과로 변동성을 줄이기 위해서인데요, 켈리기준의 시각으로 보아도 포트폴리오에 의한 분산투자효과는 매우 강력합니다.  일단 포트폴리오를 통해 서로 상관관계가 낮은 전략들이 섞이게되면 각각 전략들의 켈리비율도 올라가서 더 많이 베팅할 수 있는 결과가 나옵니다. 결과적으로 복리수익률 역시 개별전략들의 복리수익률을 평균한 것보다 높게 나옵니다.  선물시스템투자를 함에있어 좋은 전략은 대부분 켈리비율이 1을 넘어갑니다. 즉, 풀베팅을 해야된다는 것이지요. 즉, 개별 전략들에 있어 켈리비율의 계산은 옵션전략을 빼고는 큰 의미가 없을 수도 있습니다. 다만, 거래결과를 수익률분포에 업데이트시켜가면서 도출된 켈리비율의 변화를 관찰하여 전략이 현재 어떤 상황에 있고 이 전략을 언제 안 쓸것인지, 베팅비율을 줄일 것인지에 대한 리스크 관리차원의 유용성만 남게됩니다.  그러나 포트폴리오라면 이야기가 다릅니다. 포트폴리오에 포함된 개별 전략의 켈리비율이 모두 1을 넘더라도 포트폴리오의 복리수익률을 극대화하기 위해 개별 전략들 간의 한정된 자금을 어떻게 분배하여 투자할 것이냐하는 자금분배 문제가 남기 때문입니다. 최적화된 자금분배가 이루어진 포트폴리오는 단순히 상관관계만을 고려하여 각 전략간에 자금을 배분한 포트폴리오와는 차원이 다른 복리수익률을 보여줄 수 있습니다. 경우에 따라서는 포트폴리오에 포함된 개별전략의 최대복리수익률을 넘어서는 결과도 가능합니다. (최대복리수익률을 가진 개별전략의 분산이 큰 경우 이런 일이 발생합니다.) 즉, 최대 포트폴리오 복리수익률을 위해서는 단순히 상관관계만을 고려해서는 안되고 보다 정밀한 계산으로 포트폴리오의 복리수익률을 극대화하는 수리적인 스킬이 필요합니다. 사실 지금부터 나올 자금배분 시스템의 실제 구현은 간단하지는 않고 고려해야할 사항이 많기 때문에 이 글만으로는 실제적인 도움이 안되실 수가 있지만 개념을 알고 계신다면 언젠가는 분명 도움이 되실것입니다.  ----------------------------------------------------------------------------------------------------------------우선 간단한 버전입니다.  전략1, 전략2, ..., 전략n 에 각기 f1, f2, ..., fn 의 비율로 자금을 배분한다고 하면, f1+f2+...+fn = 1 이 될것입니다. 벡터 F' = (f1, f2, ..., fn) 입니다. ('는 Transpose 연산) 또 전략1, 전략2, ..., 전략n의 평균수익률을 각각 m1, m2, ..., mn 이라고 하고 벡터 M=(m1, m2, m3, ..., mn)' 입니다. C는 이 전략 수익률의 covariance 행렬이라고 하면, 수익률 = F'M   분산 = F'CF  입니다. 극대화문제를 풀면,  포트폴리오 켈리비율벡터 F* = (C^-1)M이고 이때의 장기복리성장률은 ((F*)')C(F*)/2 로 최대가 됩니다.(켈리비율벡터 F* 를 보면, 1차원일때 수익률/분산 이 됨을 알 수 있습니다.)---------------------------------------------------------------------------------------------------------------- 데이터에 입각한 정확한 수치해석방법은 다음과 같습니다. 위의 간단한 버전보다 당연히 정확합니다.  우리에게는 다음과 같은 t개의 거래데이터가 주어져 있을 것입니다. 전략1의 거래내역: X11, X12, X13, ..., X1t전략2의 거래내역: X21, X22, X23, ..., X2t...전략n의 거래내역:Xn1, Xn2, Xn3, ..., Xnt 이제 Y1 = (X11, X21, ..., Xn1)Y2 = (X12, X22, ..., Xn2)...Yt = (X1t, X2t, ..., Xnt)인 벡터 Y를 생각합니다. 포트폴리오의 복리수익률 극대화의 문제는  F = (f1, f2, ..., fn) 이라 할 때, (편의상 위의 예랑 F형태가 transpose 되어 있습니다.)argmax(F) E log (1+XF')  s.t. |f1|+|f2|+...+|fn| <= 1 (절대값인 이유는 선물이기떄문에 매도해도 프리미엄을 받을 수 없기때문입니다.) 즉 E log (1+ f1*X1 + f2*X2 + ... + fn*Xn) 의 문제를 풀어야되는 것이지요. 단일전략 켈리문제를 iteration 으로 푼 방식을 그래도 적용하면,E log (1+XF') = (1/m) * [log(1+Y1*F') + log(1+Y2*F') + ... + log(1+Yt*F')] 이므로log(1+Y1*F') + log(1+Y2*F') + ... + log(1+Yt*F') 의 극대화문제를 풀면됩니다. 단순대입법으로 풀 경우 각 f를 0.01 씩 변화시킨다고 가정할때 N개의 전략에서 대략 100^(N-1) 번의 iteration 이 필요하네요.시간이 꽤 오래걸릴듯 싶습니다. 다행히 수치해석학에서 훨씬 빨리 최적화시킬 수 있는 방법을 이미 개발해놓았습니다만 설명하려면 복잡하므로 여기서는 생략하겠습니다. Interior-Point Optimization 이나 Sequential Quadratic Programmming 등을 찾아보세요. 이러한 방식으로 최적 포트폴리오를 구하면 공분산 행렬을 구하는 작업이 위의 최대화 계산에 자연스럽게 녹아들어가게되고 훨씬 정확해집니다. 또한 앞서 글에서 이야기한 적응적 켈리비율계산법과 같이 Bayesian 접근법으로 지속적인 거래데이터를 통해 매일, 혹은 매주 거래데이터를 수익률분포에 업데이트하면 포트폴리오의 비율을 동적으로 정확하게 관리할 수 있게 됩니다. 개별 전략의 수익률이 안좋아질경우, 상관관계나 공분산이 불리하게 흘러갈 경우 자동으로 포트폴리오에서 해당 전략의 비중이 줄어들게 되므로 리스크 관리도 자동적으로 됩니다.  또다른 유용성은 자신이 보유하고 있는 전략 중 어떤 전략을 사용해야 하나라는 의문에 대한 해답을 준다는 것입니다. 자신이 가지고 있는 모든 전략(쓰레기전략 포함)을 모두 탈탈 털어서 포트폴리오 복리수익률 최적화를 돌린다면 쓰지 말아야할 전략은 자연스럽게 그 비율이 엄청나게 작게 나와서 1계약도 베팅할 수 없는 결과가 되고 0 또는 마이너스 값이 나오기도 합니다.(마이너스 값은 그 전략과 반대로 베팅하라는 의미입니다. 이때는 매수/매도를 뒤집어서 슬리피지/수수료 고려하고 다시 포트폴리오 최적화를 돌리면 됩니다. 보통 몹쓸 전략으로 판명되지요.)즉, 자기가 어떤 전략을 사용해야 될지 자연스럽게 알게 되는 것이지요.           투표는 표시되지 않습니다.  #켈리  #켈리공식  #켈리기준  #Kelly  #KellyCriterion  #자금관리  #moneymanagement  #시스템트레이딩
